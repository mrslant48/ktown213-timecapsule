Absolutely the best video/chat program I've used tho it has it's bugs.

First the good points:

The video/audio to my knowledge is streamed peer-to-peer. What this means is that you guys won't have to pay for the large amounts of video and audio data bandwidth the chat system uses.

It's certainly the most fun. The two most popular places for korean's to chat and mingle that I know of are AsianAvenue.com and Yahoo. AA doesn't have chat or voice. Yahoo has both but the way they handle webcams are that you usually have to have someone grant you permission to view their cam. Which for me often ment sucking up to some girl i don't know to just to see what the hell they look like. Avchat is a lot more fun in that you actually find a lot of people with cam's all chatting in the same room.

Now the bugs..

Unless they've fixed it in recent releases, avchat has some security flaws. The chat portion of the program actually uses IE. They parse the input field so that normally you won't be able to put in stuff like stylesheets, java, etc. but sneaky users will be able to get around this.

What I would strongly suggest is to have some trusted administrators to actually firewall IP's of abusive users from the chat. I'm not sure about how the program works server side, but I suspect that it may only run on NT/2000/XP so watch out for that if you are linux/unix based. Also it may entirely be possible that the server provides for banning IP's but otherwise it will have to manually be done at the OS or firewall.

Also I've noticed that for whatever reason the chat system seems to crash every once in a while. You may want to look out for this and have a script to restart the service if it is down automatically.

Other than that, I think it's a great idea. Very fun chat system. So you guys can get it? I think someone told me that the licensing for the software was somewhat expensive.

Well, I looked at alot of those PHP-based chat scripts, but I don't like any of them for several reasons.

1. They tend to lag. I hate the fact that you type up your message, press enter, and it takes a second for it to load.
2. Haven't found one with the "private chat" feature.
3. Free chat services are free for a reason, because they're very limited in what you can do, as an admin or a user.

Some of the advantages of using the very-expensive-saram.net-esque chat service.

1. Most people are familiar with the saram chat interface, and they'll feel right at home using it.
2. It's fast. Type up a message, press enter, and it's instantaneous.
3. Private chat feature is added.
4. Very powerful administrative tool.


Basically, I think advantage #1 is pretty significant. It's brand recognition...I can already see people going around telling their friends "Go checkout Ktown213's chat room, it's just like saram.net."

But, if you can find something truly better, I'm alllllllll for it.

Here's some of the features...

Personal Chat Rooms
Personal Chat Rooms take the visitor directly into a personal chat room unique to its Web page and not present by default in the list of public chat rooms available in the Chat server. This room has all the functionality of a regular chat room (pull-down menus, banner advertisements, etc.), and can be completely customized by defining over 170 properties as applet parameters on the Web page -- overriding the default applet properties' file.

Moderated Live Events
Moderated Events are used to host live events in an auditorium fashion with moderators, guest speakers, and an audience. All questions and comments from the audience can be filtered by the event moderators before being posted to the room.

Membership Database Support
VolanoChatPro has external database connectivity and can support any database. If you want to reserve members' names, or provide more secure authentication, the Chat server invokes 3 simple scripts that you would write to look into your database.

Chat Message Transcripts
Need to keep a log of the messages that occur in the chat rooms? The chat message logging support allows you to transcribe the conversations in your public and private chat rooms so that they're available for later use, such as posting the transcripts of a moderated event or interview to your Web site.

Membership-based Monitors
This features lets you identify specific members to be monitors, each with their own password rather than using the system-wide monitor password.

Server-side Event Reporting
You can write CGI scripts (in any programming language) or Java servlets which register with the Chat server for event notifications, for example, entering/exiting rooms, chat messages, monitor actions, etc.
